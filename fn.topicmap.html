Class: topicmap

topicmap

new topicmap(options)

Namespace and static initializer for the topicmap. Note that it's written as a jQuery plugin, so it's initialized by calling e.g. 

 $('#paper').topicmap({})
to create an instance and the methods are called using e.g. 
 $('#paper').topicmap('clear')
See topicmap-sample.html/topicmap-require-sample.js in the sample project as a minimal example, or topicmap.html/topicmap-require.html as an example of integration with a working backend.

Optional cutoff point for the minimum number of pixels a node must occupy. Any nodes smaller than this will be discarded.

If set, any text labels which don't fit in their containing nodes will be removed.
hideLegend boolean <optional>
 false

If set, the positive-negative green-to-red sentiment legend will be hidden, even if the clusterParam parameter to renderData() was true.
skipAnimation boolean <optional>
 false

If set, animations will be skipped. There may be a noticeable pause if we skip animation, since javascript is single threaded and it'll take a while to compute the final positions. The browser may also give a warning if it takes too long.

Optional callback which will be called after the treemap is configured, can be used to set treemap layout options e.g. node sort order.
onNodeRender external:jQuery.external:fn.topicmap~onNodeRender <optional>

Optional callback which will be called after a node is rendered.
showVertices boolean <optional>
 false

Debug flag. If set, markers will be drawn on the vertices during animation.
singleStep boolean <optional>
 false

Debug flag. If set, we only perform a single step of animation after rendering. You can use this with the animate() method on the plugin to start/stop/step-through animation.
onVertexHover external:jQuery.external:fn.topicmap~onVertexHover <optional>

Debug callback; will be called when the mouse hovers over a vertex. Note: vertices are only visible+hoverable when the 'showVertices' option is set true.

renderData(json, clusterSentiment)

Render tree data on the topic map.

Parameters:
Name Type Description
json external:jQuery.external:fn.topicmap~Node

The tree data to render. Should have a root node, with children nodes. Each node should have a 'name' label, a 'size' scaling factor and optionally 'sentiment' information between 0 and 1, where 0 is negative and 1 is positive.
clusterSentiment boolean

If true, nodes will be coloured based on their sentiment values and the sentiment legend will be shown.
